# Containers

An alternative way to segment a single machine into multiple virtual machines is to use _containers_. With a hypervisor, hardware is segmented to allow multiple guest operating systems to run. With containers, the host operating system is segmented into multiple containers, each running a virtual machine. Each VM thus shares a single type and instance of the operating system.  

Containers are much more economical in system resources than VMs, but they do have their downsides.
